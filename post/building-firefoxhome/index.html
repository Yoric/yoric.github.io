Announcing Project Building Firefox@home

@David Teller · Apr 1, 2020 · 2 min read

Problem statement

Project Folding@Home uses the spare CPU & GPU capabilities of a million users to simulate protein folding, something that can be useful to understand diseases and fight them.

Now, these days, Folding@Home has more users than it has server capacity to distribute and collect samples. This is where project BuildFirefox@Home comes in!

Project BuildFirefox@Home attempts to solve a pain point that many of our individual contributors encounter: building Firefox is slow. To do this, we just need to use the unused computational power of Folding@Home users.

Outline

  1. Mozilla joins the Folding@Home consortium.
  2. Mozilla offers server capacity to the Folding@Home consortium.
  3. Whenever a Firefox contributor attempts to build Firefox, their computer actually transpiles the build into a set of protein foldings. This is a simple problem of biological computing, which is left as an exercise to the reader. The contributor’s computer becomes a temporary Folding@Home server, proxified through Mozilla’s Folding@Home servers.
  4. Whenever a user of Folding@Home downloads a sample from Mozilla’s Folding@Home servers, it means that the server capacities of Folding@Home are exceeded, so we can route the request to the contributor’s personal computer, which will distribute slices of protein folding encoding their build.
  5. Whenever the slice is folded, the results are uploaded to Mozilla’s server, which serves again as a proxy for the individual contributor’s temporary server. The contributor’s computer therefore receives the slice.
  6. Once all slices are folded, the build is complete. Temporary server disconnects. Contributor is happy.
